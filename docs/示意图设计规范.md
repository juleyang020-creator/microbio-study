# 教学示意图（SVG）设计规范

本规范由 2026-07 采纳的机制图（`img/mechanism-*.svg`）提炼而成，用于统一全部教学示意图的
版式与配色。**新增或重构任何示意图都应遵循本规范，并通过 `node tools/lint-svg.mjs` 检查。**

范本文件：`img/mechanism-folate.svg`（结构最清晰）、`img/mechanism-cellwall.svg`（多环节编号）、
`img/mechanism-membrane.svg`（左右分栏对比）。

---

## 一、画布

| 项 | 规定 |
|---|---|
| viewBox | `0 0 480 300`；内容确实放不下才加高（如 `480 406`），**宽度一律 480** |
| 宽度为何固定 480 | 手机正文可用宽约 347px，480 缩放后字号约为原值 0.72 倍；宽度越大字越小。**禁止使用 720 宽** |
| 背景 | 不画背景矩形，由页面提供白底 |

## 二、字号（`tools/lint-svg.mjs` 强制下限 9.5）

| 用途 | 字号 | 字重 | 颜色 |
|---|---|---|---|
| 主标题 | 16 | 700 | `#1f2733` |
| 右上角序号徽标 | 9.5 | — | `#6b7684`（底 `#eef1f5` 圆角矩形）|
| 副标题（一句话讲清机制） | 11.5 | — | `#5c6672` |
| 图内标签 / 注解 | 9.5 | — | `#5c6672`（次要）、`#33404f`（正文）|
| 结论句（图下方总结） | 11.5 | — | `#33404f` |
| 「代表药」底栏 | 9.5 | 标签 700 | 标签 `#33404f`，内容 `#5c6672` |

**不得使用小于 9.5 的字号**——480 宽画布在手机上已缩到 0.72 倍，9.5 已接近可读下限。

## 三、固定版式（自上而下）

```
y=27    主标题（x=24, 左对齐）        右上角徽标（x=456, text-anchor=end）
y=46    副标题（x=24）
y=58    分隔线 <line x1=24 x2=456 stroke="#e7e4dd">
y=70~250 主体图形区
y≈262   结论句（x=240, text-anchor=middle, fs 11.5）
y=270~292「代表药」底栏：圆角矩形 x=24 w=432 h=24 fill="#f5f7f9" stroke="#e3e7ec"
```

## 五、内容要求（这是重点，不只是好看）

1. **一图讲一条主线**：副标题必须能用一句话说清「谁作用在哪、结果是什么」。
2. **多靶点必须分别画出并编号**。反例：早期 `cellwall` 副标题写了三类药，图上却只画 PBP 一个靶点，
   学生无法看出万古霉素结合的是底物末端、磷霉素作用在胞浆内。
3. **不同菌型/部位不可混用同一结构**。反例：早期 `membrane` 用一张通用膜同时表示多黏菌素
   （革兰阴性外膜 LPS）与达托霉素（革兰阳性胞浆膜）。
4. **图形与文字必须自洽**。反例：图上画了内膜，文字却从「外膜崩解」直接推出「内容物外漏」——
   胞浆内容物在内膜内侧，自相矛盾。
5. **可加一行临床注解**（如「被肺表面活性物质灭活 → 不用于肺炎」），教学价值高。
6. **不写死数量**（「共 32 项」这类会随数据更新过期）。徽标也不要写「6/6」。
7. **图形必须承担信息**：把所有形状删掉后若文字仍能独立读懂，说明画成了文字幻灯片，不合格。
8. **不要硬套「阴性 / 阳性」二元框架**。反例：TSI/KIA 本是「斜面/底层」四型判读（A/A、K/A、K/K…），
   早期版本强行标成「阴性：底层不产酸 / 阳性：底层产酸」，既不符合该试验的判读方式，也掩盖了
   「仅发酵葡萄糖的志贺/沙门菌斜面同样复红」这一要点。
9. **配色例外**：当颜色本身就是教学点时（麦康凯粉红菌落、TSI 黄/红/黑、甲苯胺蓝板的蓝与玫瑰粉），
   用试剂/菌落的真实颜色，不受第四节调色板限制。

## 五之二、内容正确性的工作流

示意图里的机制陈述必须独立事实核查，**不能因为「是从 `data/` 里抄的」就默认正确**——
本轮重构反向查出了数据文件自身的 3 处错误（Ehrlich 试剂误作对二甲氨基肉桂醛、
半胱氨酸产 H₂S 误称「还原」、TSI 的 H₂S 误归「亚硫酸盐还原酶」）。

同样重要的是：**审查者提出的「更正」本身也可能是错的**，必须独立复核后再落笔。
反例：TSI 斜面复红，有审查者主张把「氧化脱羧产碱」改为「氧化脱氨产氨」，
但 ASM、MacFaddin、HiMedia 等主流文献原文确实写作 oxidative decarboxylation——
这是真实的文献分歧，示意图不应选边，最终改用中性表述「氧化分解蛋白胨产碱」。

## 六、无障碍与技术

- 必须有 `<title>`（读屏与图片加载失败时的替代文本）。
- 必须有 `viewBox`，不要写死 `width`/`height` 属性。
- 注意：**带 viewBox 的 SVG 没有固有尺寸**，`width:auto` 会被拉满容器宽度，`max-width` 对其无效；
  若需在 JS/CSS 中控制显示尺寸，须由 `height` 驱动（见 `.zoom-overlay.rotate .zoom-img`）。

## 七、提交前必须通过

```bash
node tools/release-check.mjs     # 内容自检 + 示意图检查 + 预缓存清单 + 测试 + 空白字符
```

单独运行时：

```bash
node tools/lint-svg.mjs          # 文字重叠 / 超出画布 / 字号下限（有错误时退出码 1）
node tools/sync-sw-images.mjs    # 新增/删除示意图后同步 sw.js 的离线预缓存清单
```

**新增示意图的接线**：把文件名加进 `js/view.js` 对应的 `TEST_IMAGE` / `STAIN_IMAGE` /
`BIOCHEM_IMAGE` / `MEDIA_IMAGE` 映射，再跑 `node tools/sync-sw-images.mjs`。
`tests/data-integrity.test.js` 会校验映射里的文件确实存在。
