# 菌种界面编辑规范

> 本规范定义"一个微生物详情页应该包含哪些内容、内容写在哪个文件里"。
> **以「铜绿假单胞菌（pseudomonas-aeruginosa）」为黄金模板**——它是目前内容最完善的条目。
> 新增/完善任何菌种时，对照本规范逐项补齐即可，无需每次重新摸索。

---

## 0. 一个完整详情页长什么样

打开一个菌的详情页（`#/microbes/<id>`），从上到下依次渲染这些区块（见 `js/app.js` `buildDetail`）：

| 顺序 | 区块 | 数据来源文件 | 是否必填 |
|---|---|---|---|
| — | 天然耐药（红色提示条） | `data/microbes.js` 的 `天然耐药` 字段 | 视菌而定 |
| — | 形态与染色 / 致病性（文字小节） | `data/microbes.js` 的 `小节` | **必填** |
| ① | **形态**（镜下 + 各培养基形态） | `data/morphology.js` | 推荐 |
| ② | **折点**（CLSI 药敏表） | `data/breakpoints.js`（把 id 加进菌组的 `菌种`） | 见 §5 |
| ③ | **相似菌与鉴别** | `data/differential.js` | 推荐 |
| ④ | **生化反应**（可点链到生化试验） | `data/biochem.js` | 推荐 |
| — | 关联（可跳转的相关条目） | `data/microbes.js` 的 `关联` | **必填** |

> **"完善"= 这 5 张表都为该 id 写了内容**（铜绿假单胞菌就是如此）。
> 只写 `microbes.js` 不写另外四个文件，详情页就只有两行字——这是之前"人葡萄球菌那种"的根因。

所有文件都用 `id`（如 `pseudomonas-aeruginosa`）作为主键，**五处的 id 必须完全一致**。

---

## 1. `data/microbes.js` —— 主条目（必填）

```js
{
  id: 'pseudomonas-aeruginosa',                 // 唯一，kebab-case，全库一致
  天然耐药: '对氨苄西林、阿莫西林/克拉维酸、第一/二代头孢、头孢曲松、厄他培南、四环素类、复方磺胺、氯霉素天然耐药；仅抗假单胞药物有效。',
  名称: '铜绿假单胞菌',
  拉丁名: 'Pseudomonas aeruginosa',
  类别: '假单胞菌属',                            // 必须是 categories.js 里存在的叶子节点
  小节: [
    { 标题: '形态与染色', 正文: '革兰阴性杆菌，专性需氧，产蓝绿色素与生姜气味；单端鞭毛。' },
    { 标题: '致病性 / 所致疾病', 正文: '院内感染、呼吸机相关肺炎、烧伤创面/绿脓、中性粒细胞缺乏者感染；天然多重耐药。' }
  ],
  关联: ['ceftazidime', 'piperacillin-tazobactam', 'tobramycin', 'ciprofloxacin', 'meropenem']
}
```

规则：
- **`小节` 只放两节**：`形态与染色`（一句话：革兰染色+形态+关键快速试验）和 `致病性 / 所致疾病`（一句话）。
  细节（镜下/培养、生化、鉴别）**不要堆在小节里**，放到 §2–§4 的专用文件，否则风格突兀。
- `天然耐药`：只有"该菌对某些药**先天**无效"时才写（铜绿、不动杆菌、黄杆菌等）；普通菌（金葡、头葡）留空、不写该字段。
- `关联`：3–5 个，填**首选/代表性抗菌药 id** 或**相关菌 id**；id 必须真实存在（`validateData` 会校验）。
- `类别` 必须能在 `data/categories.js` 找到，否则 `validateData` 报错。

---

## 2. `data/morphology.js` —— 形态 ① （推荐）

`window.DB.morphology` 是 `{ id: { 镜下, 培养:[{培养基,形态}] } }`：

```js
'pseudomonas-aeruginosa': {
  镜下: '革兰阴性杆菌，单端鞭毛。',
  培养: [
    { 培养基: '血平板', 形态: '灰绿色、扁平、金属光泽菌落，产绿脓菌素(蓝绿)、生姜气味，常 β 溶血。' },
    { 培养基: '麦康凯', 形态: '不发酵乳糖→无色菌落。' }
  ]
}
```

- `镜下`：革兰染色、形态、排列、鞭毛/芽孢/荚膜。
- `培养`：每个常用培养基一行；常用 `血平板`、`麦康凯`、`巧克力`、`SS`、`甘露醇高盐琼脂`、`沙保罗` 等。

---

## 3. `data/differential.js` —— 相似菌与鉴别 ③ （推荐）

`window.DB.differential` 是 `{ id: [ {名称, id?, 相似点, 鉴别} ] }`：

```js
'pseudomonas-aeruginosa': [
  { 名称: '鲍曼不动杆菌', id: 'acinetobacter-baumannii',
    相似点: '均为非发酵革兰阴性杆菌、院内多重耐药',
    鉴别: '氧化酶：铜绿(+)、不动杆菌(−)；铜绿产绿脓菌素、有动力。' }
]
```

- 每个易混菌一项；`id` 填相似菌的 id（可点击跳转），没有对应条目可省略 `id`。
- `鉴别` 要给**可操作的区分点**（点名具体反应/表型），不要泛泛而谈。

---

## 4. `data/biochem.js` —— 生化反应 ④ （推荐）

`window.DB.biochem` 是 `{ id: [ {项目, 结果} ] }`：

```js
'pseudomonas-aeruginosa': [
  { 项目: '氧化酶', 结果: '+' },
  { 项目: '乳糖发酵', 结果: '−（非发酵）' },
  { 项目: '绿脓菌素', 结果: '+' },
  { 项目: '42℃ 生长', 结果: '+' },
  { 项目: '动力', 结果: '+' }
]
```

- 挑**最有鉴别意义**的反应（氧化酶/触酶/凝固酶/动力/吲哚/脲酶/O-F/七叶苷/PYR/糖发酵…），阴阳务必正确。
- `项目` 用标准试验名：若与 `data/biochem-tests.js` 里的试验同名，详情页会**自动变成可点击链接**跳到该试验原理页。
- 同名 `项目` 还能让多个菌在"生化反应对比"里对齐成一行。

---

## 5. `data/breakpoints.js` —— 药敏折点 ② （按标准而定）

**药敏不写在菌条目里**，而是把菌的 id 加进对应 CLSI 菌组的 `菌种` 数组，详情页就会自动渲染该组的 M100 折点表（铜绿在"铜绿假单胞菌 2B-1"组里）。

```js
{ 菌组名: "不动杆菌属 (Acinetobacter spp.)", CLSI表: "Table 2B-2",
  菌种: ["acinetobacter-baumannii", "acinetobacter-pittii", "..."],   // ← 把新 id 加这里
  药物: [ ... ] }
```

### 5.1 该挂到哪个组（现有 M100/M45/M60 菌组）
- 肠杆菌目（大肠/克雷伯/变形/沙雷/哈夫尼亚/拉乌尔/普罗威登斯…）→ **2A-1**
- 沙门/志贺 → 2A-2；铜绿 → 2B-1；不动杆菌 → 2B-2；窄食单胞 → 2B-4；伯克霍尔德 → 2B-3
- 葡萄球菌（含所有 CoNS）→ **2C**；肠球菌 → 2D
- β-溶血链球菌 → 2H-1；草绿色/咽峡炎群链球菌 → 2H-2；肺炎链球菌 → 2G
- 嗜血杆菌 2E、淋球菌 2F、脑膜炎奈瑟菌 2I
- 厌氧菌 → 2J；李斯特/卡他莫拉/幽门螺杆菌 → CLSI M45
- 念珠菌 → M60（白念/光滑/近平滑/热带/克柔分组），耳念珠菌用 CDC 暂定

### 5.2 哪些菌"本来就没有 M100 药敏"——属于正常，不是缺漏
> ⚠️ 关键：不是所有菌都有标准折点。**没有就不要硬凑，也不要编数值。**

- **CLSI M45（少见/苛养菌，本库尚未收录该标准的完整折点）**：棒状杆菌属（**Corynebacterium**，如白喉/纹带棒杆菌）、气球菌属（Aerococcus）、乏养菌/颗粒链菌（Abiotrophia）、HACEK（金氏菌 Kingella 等）、乳杆菌、芽孢杆菌等。
- **无任何标准折点**：微球菌属（**Micrococcus**，如藤黄微球菌）、罗斯菌属（Rothia）、以及多数环境非发酵菌——鞘氨醇单胞、金黄杆菌/伊丽莎白金菌、香味菌、寡源杆菌、罗尔斯顿、代尔夫特、产碱杆菌等。

对这些菌：
- **不挂折点表**（详情页不出现 ② 折点区块，属正常）。
- 用 `关联` 给出**经验首选药**（如微球菌/棒杆菌→`vancomycin`），必要时在 `天然耐药` 写一句先天耐药。
- 形态/鉴别/生化（①③④）照常补全——这样条目依然"完善"，只是没有折点表。
- 若日后拿到 CLSI **M45/M60** 文件，可照 §5 新建对应菌组再把 id 挂进去。

## 6. 新增/完善一个菌种 —— 检查清单

1. `microbes.js`：`id / 名称 / 拉丁名 / 类别 / 小节(形态与染色+致病性) / 关联`，必要时 `天然耐药`。
2. `morphology.js`：`镜下 + 培养[]`。
3. `differential.js`：列出易混菌 + 可操作鉴别点。
4. `biochem.js`：关键生化反应（阴阳正确，项目名尽量对齐生化试验库）。
5. `breakpoints.js`：**若该菌有 M100/M45/M60 折点** → 把 id 加进对应菌组 `菌种`；**没有就跳过**（见 §5.2）。
6. 五个文件里的 `id` 完全一致。
7. 跑测试：`node --test tests/data-integrity.test.js tests/core.test.js tests/view.test.js tests/tools.test.js`（`validateData` 会校验 id/类别/关联）。
8. 升 `sw.js` 版本号（`microbio-vNN`）以刷新缓存。

> 自检："这个菌的详情页是不是和**铜绿假单胞菌**一样有 形态/折点(或合理地没有)/鉴别/生化/关联？"——是，就算完善。
